A guide for product designers

You don't need
to learn to code.
You need to learn
to direct.

Vibe coding lets product designers build real things — portfolio sites, prototypes, internal tools — by describing what they want in plain English and letting AI do the rest. This is the guide to getting started.

No coding background required. No jargon. Just a clear vision and a willingness to iterate.

01 What is vibe coding?

A new way to build.

In February 2025, Andrej Karpathy — co-founder of OpenAI, former AI lead at Tesla — posted a note about a new way he'd been writing software. He called it "vibe coding."

The idea: stop thinking about code. Describe what you want in plain English. Let an AI write the implementation. Test whether it works. Iterate with more prompts. Repeat until it's done.

Think of it like directing a film. You are not operating the camera. You're telling the cinematographer exactly what you want the shot to feel like — and they figure out how to create it.

The programming language is now English. And for designers, who already think in outcomes, systems, and user experience, that changes everything. Your design thinking is now your most valuable coding skill.

"There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."
— Andrej Karpathy, February 2, 2025  ·  Collins Dictionary Word of the Year 2025
02 Your two tools

Two tools.
One for each mode of working.

You don't need to learn twenty tools. You need two. They complement each other — one is a full editor, the other is a powerful AI assistant. Together they cover everything from first prototype to polished product.

Tool 01

Cursor

A code editor with AI built into every keystroke. Download it and it looks like any familiar text editor — except you can describe what you want and it writes the code. No terminal required, ever.

Best for

Beginners. Visual-first workflows. Precise, targeted edits. If you've never opened a terminal, start here.

Where to start

cursor.com — downloads like any app

Key insight Cursor has an "autonomy slider." You control how much freedom you give the AI — from gentle autocomplete to full agent mode, where it plans and builds entire features on its own.

Tool 02

Claude Code

Anthropic's AI coding assistant. It understands your entire project and can read files, write code, fix errors, and commit changes — all directed in plain English. Think of it as a very capable developer you can direct entirely by description.

Best for

Larger projects. Cleaning up and refactoring existing work. Working across many files at once. Complex, multi-session projects.

Where to start

claude.ai/code — works in browser, no install needed

Key insight Add a file called CLAUDE.md to your project. Write your preferences in it — fonts, design principles, structure rules. Claude reads it at the start of every session, like a creative brief.

The workflow: start in Cursor, stay there if it's going well. When you've built something worth keeping and want to clean it up or add complexity, bring in Claude Code. Or use one exclusively — both will get you there.

03 Getting started

From zero to something real.

  1. 01

    Install your tools

    Download Cursor from cursor.com. It installs like any Mac or Windows application — no terminal, no package manager, no configuration beyond signing in.

    What you do

    Go to cursor.com, click download, open the installer. Sign in with an email. Done.

    What happens next

    Cursor opens and looks like a familiar text editor with an AI chat panel on the side. You're ready to start.

    Optional: For Claude Code, go to claude.ai and sign in. The web interface works without installing anything — useful for experimenting before committing.
  2. 02

    Set up your first project folder

    Create a folder on your desktop called "my-first-project." Open Cursor. Go to File → Open Folder and select it. You now have a project.

    What you do

    Create folder → open in Cursor. That's your workspace. Every file the AI creates goes here.

    What happens next

    Cursor shows you an empty folder in the left panel. The AI chat is on the right. This is your canvas.

    Keep it simple: One folder, no subfolders. One index.html file, one style.css. The AI will create them.
  3. 03

    Write your brief, then your first prompt

    Before opening the AI chat, spend five minutes writing: What is this page for? Who is it for? What sections does it have? What does it feel like? Then paste that as your first message.

    What you type

    Build me a single-page personal portfolio. It should have: a hero section with my name and a one-line bio, a projects section with a 3-column grid of placeholder cards, and a contact section with my email. Style: minimal and typographic. Warm white background, near-black text, no images. Serif font for headings, clean sans-serif for body.

    What happens next

    Cursor generates your HTML and CSS. It shows you exactly what changed. Open the index.html file in your browser (right-click → Open With) to see your page live.

  4. 04

    Iterate with targeted prompts

    Look at your page. What's wrong? Be specific. Then tell the AI exactly what to fix. This loop — describe, accept, test, refine — is the entire practice of vibe coding.

    What you type

    The hero heading is too small. Make it 72px on desktop, 40px on mobile. The project grid collapses to 1 column on mobile. It should be 2 columns on tablet, 1 on phone. Add a smooth fade-in animation when each section enters the viewport on scroll. I got this error: [paste error]. Fix it.

    What happens next

    The AI makes the change, shows you a diff of what changed, and you accept or reject it. When something breaks — and it will — copy the error message and paste it back. That's the debugging workflow.

    Specificity is the skill. "Make it look better" produces unpredictable results. "Increase heading size to 56px, tighten line-height to 1.1" produces exactly what you want. Designers already think in precise values — use that.
  5. 05

    Deploy and share something real

    When your page is ready to share, use Netlify Drop. Drag your project folder directly onto the page. In thirty seconds, it's live at a public URL.

    What you do

    Go to app.netlify.com/drop. Drag your project folder onto the page. Wait thirty seconds.

    What happens next

    Your page is live at a public URL you can share with anyone. No accounts required for a basic deploy. No command line. No configuration.

    Custom domain: Both Netlify and Vercel support custom domains with a few clicks. Ask the AI to walk you through it.
04 First project ideas

Start here.

Project 01

Your Portfolio Page

A hero with your name and tagline, a project grid with case study links, and a contact section. You already have the content — your work, your bio, your taste. The AI just needs to build the container.

Why this first Highest motivation. Immediately useful. You know exactly what "right" looks like.
Project 02

A Single Case Study

Pick one project. Tell the story in a single scrollable page: problem, process, solution, outcome. Easier to scope than a full portfolio. A useful standalone piece for a pitch or job application.

Why this first Scoped enough to finish in an evening. Ships as something genuinely useful.
Project 03

Design System Showcase

A page that renders your color palette, type scale, spacing tokens, and component examples as a living visual reference. Useful for your own work and impressive in a portfolio. You'll learn how CSS variables work without reading a line of code.

Why this first Builds on existing design work. Teaches you how design tokens translate to code.
Project 04

A Tool for Your Own Workflow

A client brief generator. A proposal template form. A link-in-bio page. A mood board grid. Something you'd actually use daily. The more personal the use case, the better your prompt quality — because you know exactly what "right" looks like.

Why this first Personal motivation is the strongest force in vibe coding. Build for yourself first.
05 Mindset & tips

What separates
the ones who ship.

  1. 01

    Be the director, not the developer.

    Your job is to have a clear vision and articulate it precisely. The AI's job is to implement. Strong communication is the actual skill here — not syntax, not package managers, not debugging. You're already good at this. Clear briefs produce clear output.

  2. 02

    Specificity is everything.

    "Make it look better" tells the AI nothing. "Increase the heading size to 56px, tighten line-height to 1.1, and change the color to #0F0F0E" tells it exactly what you want. Designers already think in precise values — font sizes, spacing scales, hex codes. That precision is your superpower. Use it.

  3. 03

    Errors are the workflow, not failures.

    When something breaks — and it will — copy the error message verbatim and paste it back into the chat. That's the debugging workflow. You don't need to understand the error. You don't need to read the code. Just paste and ask for a fix. A vibe coding session is a series of prompt → output → test → fix loops. That is how it works.

  4. 04

    Build one thing at a time.

    Don't ask for the full page in one prompt if you can help it. Build the hero. Confirm it works. Build the next section. The more scoped your prompts, the cleaner the output — and the easier it is to course-correct when something isn't right. Think in components, exactly as you would in Figma.

  5. 05

    Ship at 80%. Iterate from there.

    The designers who finish their first project are the ones who resist perfecting it before it's alive. Deploy the 80% version. Get it in front of someone. Then improve it. Momentum is a real force. An imperfect page on the internet is infinitely more useful than a perfect page that never shipped.
